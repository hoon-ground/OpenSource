# Goal

Flask 기반 Dev Log 웹 서비스에 대해 pytest를 활용하여 TDD(Test-Driven Development) 방식으로 테스트를 작성하고 기능을 구현한다.

# Steps

1. 메인 페이지("/")가 정상적으로 렌더링되는지 테스트 작성
2. 기록 작성 페이지("/write")에 대한 GET 요청 테스트 작성
3. 기록 작성 POST 요청 시 데이터가 정상적으로 저장되는지 테스트 작성
4. 저장된 기록이 메인 페이지 목록에 표시되는지 검증
5. 기록 상세 페이지("/log/<id>")가 정상적으로 동작하는지 테스트 작성
6. 잘못된 입력(빈 제목/내용)에 대한 유효성 검증 테스트 작성
7. 모든 테스트를 통과하도록 Flask 코드를 구현 및 수정

# Constraints

* Flask의 test_client를 사용하여 요청을 테스트할 것
* pytest 프레임워크를 사용할 것
* 단순 상태 코드 확인이 아니라, 실제 데이터 변화(logs 리스트)를 검증할 것
* 테스트 간 독립성을 위해 상태 초기화(logs.clear())를 수행할 것
* 코드 가독성과 유지보수성을 고려하여 테스트를 작성할 것
* TDD 사이클 (RED → GREEN)을 명확하게 따를 것

# TDD Workflow

* RED: 테스트를 먼저 작성하고 실패 상태를 확인한다
* GREEN: 테스트를 통과하도록 최소한의 코드를 구현한다
* REFACTOR: 코드 구조를 개선하면서 테스트는 유지한다

# Expected Outcome

* 주요 웹 기능(조회, 작성, 상세보기, 유효성 검증)에 대한 테스트 코드 완성
* 모든 테스트가 통과하는 안정적인 Flask 애플리케이션 구현
* TDD 기반 개발 흐름과 AI 협업 과정이 명확히 드러나는 결과물
